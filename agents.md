# AI Agent 工作规范

## 临时文件与测试脚本管理规则
为了保持项目目录的整洁，请你在编写测试脚本、执行验证或输出临时结果时，严格遵守以下规则：

1. **专属沙箱**：绝对不允许在项目根目录或业务代码文件夹中创建临时文件。所有的测试脚本、抓取结果、日志和临时 JSON/TXT 文件，**必须**统一保存在 `.ai_scratch/` 目录下。跑 backend 的 Python 命令时使用非沙箱（提权）执行
2. **命名规范**：所有你创建的临时脚本，文件名必须以 `agent_test_` 或 `tmp_` 开头，例如 `agent_test_db_conn.py`。
3. **阅后即焚**：如果你只是为了验证某个 API 是否连通、或者测试一小段逻辑，在得出结论并向我汇报后，请**主动删除**该测试脚本和产生的输出文件。
4. **长效测试**：如果某个测试脚本具有复用价值（例如复杂的环境检查），请在脚本顶部加上注释说明其用途，并保留在 `.ai_scratch/` 目录中，不要删除。

## 后端 Python 执行补充规则（新增）
1. 涉及 `backend/.venv/Scripts/python.exe`、`start.ps1`、`alembic`、`pytest`、`pip` 的命令，默认必须使用**非沙箱（提权）**执行。
2. 不允许仅基于沙箱中的 Python/venv 报错（如 `No Python at ...`）直接下结论；必须先提权复核一次再判断。
3. 若提权后可正常执行，以提权结果为准；若仍失败，再给出修复建议。

## 测试数据清理规则（新增）
1. 只要测试流程会写入数据库（例如创建房间、创建任务模板、写消息），就必须在测试结束后执行清理，禁止留下测试数据。
2. 清理范围必须覆盖关联数据：`rooms`、`room_members`、`messages`、以及无引用的测试 `tasks`（如 `E2E*` / `Diag*`）。
3. 默认使用 `.ai_scratch/agent_test_cleanup_rooms.py` 做前后双清理；运行会写库测试时，优先通过 `.ai_scratch/agent_test_run_with_room_cleanup.ps1` 执行。
4. 若当次只是临时排查，也必须在汇报前完成清理并复检（输出 `NO_MATCHING_TEST_DATA` 或等价结果）。
